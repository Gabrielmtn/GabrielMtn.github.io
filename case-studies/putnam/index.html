The fundamental
Build the consistency keyboard users need, first.
Working principle

This was a full accessibility refactor for Putnam Investments. Dozens of interactive components (comboboxes, menus, dialogs, disclosure widgets) needed to be rebuilt from the ground up to meet WCAG AA. I worked directly from WCAG keyboard interaction patterns to build each component.

The work taught me to think like a screen reader user: keyboard first. Navigation happens with arrow keys, Enter, Escape, and Tab. I implemented keyboard behavior before adding ARIA roles to communicate state. Every component was validated in JAWS, NVDA, and VoiceOver before shipping.

The anatomy

Building a combobox from WCAG keyboard patterns.

The WCAG keyboard interaction documentation made the patterns clear: arrow keys navigate options, Enter selects, Escape closes and returns focus. This spec shows how I translated that into a working component.

Component spec derived from WCAG keyboard interaction patterns and validated against JAWS, NVDA, and VoiceOver before ship. One of dozens in the Putnam refactor set.

The work

Building from WCAG patterns, validating with real users.

  1. Started with WCAG keyboard interaction documentation for each component type. Implemented arrow key navigation, Enter/Escape behavior, and focus management exactly as the patterns specified. Added ARIA roles to communicate state to screen readers after the keyboard behavior was correct.
  2. Treated page structure as a navigation system. Landmark regions, heading hierarchy, and tab order had to work together. A user's mental map depends on all three being consistent. Fix one in isolation and the others break.
  3. Made focus management part of the component design. Dialogs return focus to the trigger on close. Composite widgets use roving tabindex. Modal focus traps are intentional and escapable. Tested every pattern by navigating without a mouse before adding screen reader validation.
  4. Tested every component in JAWS, NVDA, and VoiceOver before shipping. If the keyboard interaction was predictable, screen readers announced it correctly. If keyboard behavior was wrong, no amount of ARIA could fix it.
